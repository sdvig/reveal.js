Route reversal

Tim

Problem:

  • Page URLs differ between countries, but we don't want to hardcode the differences
  • There are several URLs for one type of page, depending on the amount of information we have available
// DE restaurants
/lieferservices-berlin/
/lieferservices-berlin/10713/
/lieferservices-berlin/10713/sushi-bestellen

// AU restaurants
/takeaway-sydney/
/takeaway-sydney/ultimo/
/takeaway-sydney/ultimo/sushi-delivery

// DE menu
/lieferservices-berlin/irgendein-lieferservice/1337

// AU menu
/takeaway-sydney/ultimo/some-restaurant/123
/takeaway-sydney/some-restaurant/123

Solution:

  • Automate parsing the routes, analyze structure, create functions that can fill in the details into our route URL definition
  • Find route that matches given data most closely, fill in data and return it
var routesManager = require('app/routes');
routesManager.reverse('restaurantList', {
	city: 'sydney',
	category: 'sushi'
}).url;
>> '/takeaway-sydney/sushi-delivery'

routesManager.reverse('restaurantList', {
	city: 'sydney',
	suburb: 'ultimo'
}).url;
>> '/takeaway-sydney/ultimo'
